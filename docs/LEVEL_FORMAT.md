# Level Format

Формат файлов уровней в `assets/levels/`.

## Символы

| Символ | Описание | Пример |
|--------|----------|--------|
| `#` | Стена (непроницаемая) | `########` |
| `P` | Игрок (начальная позиция) | `P` |
| `B` | Ящик (который нужно толкать) | `B` |
| `T` | Целевая позиция (для ящика) | `T` |
| ` ` (пробел) | Пустое место / пол | `   ` |

## Пример уровня

```
########
#      #
# P  T #
#  B   #
#   T  #
#  B   #
#      #
########
```

**Описание:**
- Уровень 8x7 (ширина x высота)
- 1 игрок (`P`)
- 2 ящика (`B`)
- 2 цели (`T`)
- Стены по периметру

## Правила создания уровней

### 1. Обязательные элементы

- **Ровно один игрок** (`P`) — если больше или нет, уровень не загрузится корректно
- **Хотя бы один ящик** (`B`)
- **Хотя бы одна цель** (`T`)
- **Количество ящиков = количеству целей** — проверяется на этапе компиляции

### 2. Размеры

- **Прямоугольная сетка** — все строки должны быть одинаковой длины (короткие дополняются пробелами)
- **Рекомендуемый размер**: 8x7 до 15x12 для комфортной игры
- **Ограничение**: уровень должен помещаться на экране (800x600, тайл 64px)

### 3. Стены

- **Замкнутый периметр** — игрок не должен выходить за пределы уровня
- **Внутренние стены** — допустимы для создания препятствий

### 4. Проверяемость

Перед добавлением уровня проверь:
- [ ] Все ящики можно поставить на цели
- [ ] Нет "запертых" ящиков (которые нельзя сдвинуть)
- [ ] Игрок может достичь всех ящиков

## Как добавить новый уровень

1. **Создай файл** `assets/levels/level_XX.txt` (где XX — номер уровня)
2. **Нарисуй уровень** текстовым редактором используя символы выше
3. **Собери проект** — `cargo run` (уровень встроится в бинарник при компиляции)
4. **Проверь в игре** — пройди уровень
5. **Закоммить** изменения

> **Примечание:** Уровни встраиваются в бинарный файл через `build.rs`. При изменении `.txt` файлов проект автоматически пересобирается.

## Примеры

### Простой уровень (3x3)
```
#####
#P T#
# B #
#   #
#####
```

### Уровень с внутренними стенами
```
########
# P  T #
# ###  #
# #B T #
# ###  #
#      #
########
```

### Сложный уровень (несколько ящиков)
```
##########
#        #
# P  BB  #
#  TTTT  #
#   BB   #
#        #
##########
```

## Отладка

### Ошибки компиляции

Если уровень не компилируется:

1. **`panic: количество ящиков не равно количеству целей`** — проверь баланс `B` и `T`
2. **`Неизвестный символ 'X'`** — используй только допустимые символы (`#`, `P`, `B`, `T`, пробел)

### Ошибки в игре

Если уровень загрузился, но играется некорректно:

1. Проверь, что все строки одинаковой длины
2. Проверь, что есть ровно один `P`
3. Проверь, что нет неизвестных символов

## Архитектура

**Встроенные ресурсы:**
- Файлы `.txt` читаются при компиляции через `build.rs`
- Генерируется `enum Tile` для каждого символа
- Уровни хранятся как `[Tile; N]` в бинарнике
- Валидация (ящики == цели) происходит на этапе сборки

**Ключевые файлы:**
| Файл | Описание |
|------|----------|
| `build.rs` | Чтение `.txt` файлов, генерация констант |
| `src/embedded.rs` | Типы `Tile`, `LevelData`, подключение данных |
| `src/systems/level.rs` | Загрузка уровня из `LevelData` |

---

*Формат используется в `src/systems/level.rs` → `load_level()`*
